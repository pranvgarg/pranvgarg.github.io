Architecting an Intelligent Procurement Assistant

At Kimberly-Clark, a significant part of my journey involved architecting and engineering an enterprise AI agent to automate and enhance procurement operations. I led the development of a GenAI-powered chatbot, supervising its creation from end to end. This process began with a deep dive into user needs, where I conducted detailed requirement analyses to ensure the final solution would be truly data-driven and effective. The core of this initiative was building a technical solution that integrated Azure OpenAI and its powerful Large Language Models (LLMs) with our existing systems. I designed a sophisticated three-layer API gateway architecture to serve as the chatbot's foundation, creating a robust and scalable system to handle a massive volume of interactions. The chatbot was engineered to serve as an intelligent assistant, automating query resolution and streamlining workflows through an intelligent routing system that was continuously fine-tuned to optimize its performance. To support global teams, I implemented multilingual NLP capabilities using Azure Cognitive Services, which allowed for dynamic entity extraction and translation. Furthermore, I designed unsupervised clustering pipelines to group supplier queries and automate category tagging, which improved routing accuracy and response consistency in ServiceNow. The project culminated in demonstrating the chatbot's value to executive audiences through a compelling proof-of-concept, showcasing its ability to transform procurement efficiency.
